GUIDE.GITHUB

GitHub 连接使用说明

只需在每台电脑首次使用时配置一次

连接信息保存在当前博客的 .tood-studio/github.json，不会被 Git 提交。仅分享 TOOD-Studio-Windows 程序文件夹时不会携带这些信息；如果复制整个博客项目，请先在发布页点击“清除本机连接”。

01

准备 GitHub 仓库

先在 GitHub 建立用于部署博客的仓库，并确认 Cloudflare Pages 或 Workers 已连接该仓库。记录仓库地址和部署分支，通常为 main。

02

创建安全 Token

  1. 打开 GitHub 的 Fine-grained token 页面。
  2. Repository access 只选择当前博客仓库。
  3. Repository permissions 中将 Contents 设置为 Read and write。
  4. 创建后立即复制 Token；GitHub 不会再次完整显示。
打开 GitHub Token 创建页面 ↗
03

填写连接信息

GitHub 仓库
填写 用户名/仓库名 或完整仓库地址。
发布分支
填写 Cloudflare 正在监听的分支，一般是 main。
提交用户名与邮箱
用于生成 Git 提交记录，可填写 GitHub 账号名称和经过验证的邮箱。
Token
首次连接必须填写；以后修改其他字段时可以留空保留。
04

验证并发布

返回发布页点击“验证并保存连接”。验证成功后，再点击“检查并发布”。程序会先执行 Hugo 构建、生成提交并同步到 GitHub，Cloudflare 随后自动部署。

05

换电脑或交给其他人

新电脑不会自动获得旧电脑的 Token，需要重新配置。不要通过聊天、邮件或网盘发送 Token；如果怀疑泄露，应立即在 GitHub 中撤销并重新创建。

06

常见错误

  • Repository not found：仓库地址错误，或 Token 未授权该仓库。
  • 403 / Resource not accessible：Contents 没有 Read and write 权限。
  • 分支不存在：发布分支与 GitHub 仓库实际分支不一致。
  • 提交身份未知：重新执行“验证并保存连接”。
